# Contributing

Thanks for your interest in contributing.

## Guidelines

- Keep changes focused and easy to review.
- Preserve existing gameplay behavior unless the change is clearly documented.
- Prefer backward-compatible changes where possible.
- Document config, command or placeholder changes in `README.md`.

## Development Notes

- Source code lives under `src/main/java`.
- Plugin resources live under `src/main/resources`.
- Release jars should be placed under `artifacts/` only for published snapshots or tagged releases.

## Pull Requests

- Describe the problem being solved.
- Describe any gameplay or config impact.
- Include verification notes when changing mine scanning, reward logic or persistence.
